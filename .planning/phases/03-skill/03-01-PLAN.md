Create 7 skeleton files for `skills/ll-experience-patch-settle/`, following the exact pattern of `skills/ll-patch-capture/`. Per D-01, this is an independent skill — do NOT extend ll-qa-settlement.

1. **ll.contract.yaml** — Set fields:
   - skill: ll-experience-patch-settle
   - version: "1.0"
   - adr: ADR-049
   - category: experience
   - chain: patch-settle
   - phase: settlement

2. **ll.lifecycle.yaml** — Set fields:
   - skill: ll-experience-patch-settle
   - version: "1.0"
   - state: draft
   - lifecycle_states: [draft, active, validated, pending_backwrite, retain_in_code, upgraded_to_src, backwritten, archived]
   These map to ADR-049 Section 6.1 lifecycle states relevant to settlement.

3. **input/contract.yaml** — Define required inputs:
   - feat_id (string, required): FEAT ID to settle patches for
   - workspace (string, required): Project root directory
   - change_class_filter (string, optional): "visual" | "interaction" | "semantic" to filter which patches to settle
   - auto_approve (boolean, optional, default: true): Per D-08, default to fully automated

4. **output/contract.yaml** — Define expected outputs per D-03, D-04, D-05, D-07:
   - resolved_patches.yaml: Settlement report recording each patch's处理方式, timestamp, operation summary
   - Delta files for interaction patches: ui-spec-delta.yaml, flow-spec-delta.yaml, test-impact-draft.yaml
   - SRC candidates for semantic patches: SRC-XXXX__{slug}.yaml
   - Updated patch_registry.json with status changes
   - No frozen SSOT files modified (D-05)

5. **input/semantic-checklist.md** — Pre-settlement validation:
   - Feat directory exists with patch_registry.json
   - At least one pending_backwrite patch found
   - All patches pass schema validation via cli/lib/patch_schema.py

6. **output/semantic-checklist.md** — Post-settlement validation:
   - resolved_patches.yaml exists and contains entries for all settled patches
   - All processed patches have status updated from pending_backwrite to terminal state
   - patch_registry.json last_updated reflects settlement time
   - No frozen SSOT files were modified

7. **SKILL.md** — Write settlement skill overview:
   - name: ll-experience-patch-settle
   - description: "ADR-049 governed skill for batch patch settlement — reads pending_backwrite patches, classifies by change_class, generates delta drafts and SRC candidates, updates statuses, and produces settlement report."
   - Canonical Authority: ssot/adr/ADR-049-引入体验修正层-Experience-Patch-Layer.md (Sections 4.4, 7, 8, 12.3)
   - Required Read Order: ll.contract.yaml → input/contract.yaml → output/contract.yaml → agents/executor.md → agents/supervisor.md → input/semantic-checklist.md → output/semantic-checklist.md
   - Execution Protocol:
     1. Accept feat_id + workspace input
     2. Scan ssot/experience-patches/{feat_id}/ for pending_backwrite patches
     3. Group patches by change_class (D-09): visual, interaction, semantic
     4. For each group, process per backwrite mapping (D-02, D-03, D-04)
     5. Generate delta/SRC content via executor agent
     6. Update patch statuses + patch_registry.json
     7. Produce resolved_patches.yaml settlement report
   - Backwrite mapping (per ADR-049 Section 4.4):
     - visual → retain_in_code (D-02): no SSOT backwrite, mark resolved
     - interaction → pending_backwrite (D-03): generate ui-spec-delta.yaml + flow-spec-delta.yaml + test-impact-draft.yaml
     - semantic → upgraded_to_src (D-04): generate SRC-XXXX__{slug}.yaml candidate
   - Escalation conditions (D-10): change_class ambiguity, test_impact uncertainty, same-file multi-patch conflicts
   - Workflow Boundary: Input = feat_id + workspace. Output = resolved_patches.yaml + delta files + updated registry. Out of scope = TESTSET联动 (Phase 4), Hook集成 (Phase 6), 24h blocking (Phase 7), actual SSOT merge (future milestone per D-05)
   - Non-Negotiable Rules:
     - Do not modify any frozen SSOT files (D-05)
     - Do not skip schema validation before processing each patch
     - Delta files MUST include original_text, proposed_change, rationale fields (D-06)
     - Default to automated processing (D-08); escalate only on D-10 conditions
  </action>
