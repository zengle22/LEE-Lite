## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| feat_id input → filesystem | Crafted feat_id could traverse outside ssot/experience-patches/ |
| Patch YAML files → processing | Malformed or malicious YAML could crash runtime or inject data |
| Registry JSON → update | Concurrent writes or partial failures could corrupt registry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Tampering | run_skill feat_id validation | mitigate | Regex `^[a-zA-Z0-9][\w.\-]*$` + `resolve().startswith(base_dir.resolve())` path containment check, identical to patch_capture_runtime.py lines 143-158 |
| T-03-05 | Tampering | scan_pending_patches YAML loading | mitigate | `yaml.safe_load()` only (never yaml.load()); wrap in try/except to skip malformed files; validate via `patch_schema.py validate_file()` before processing |
| T-03-06 | Tampering | update_registry_statuses JSON write | mitigate | Read entire registry into memory, update in-memory dict, single `json.dump()` call at end — if any step fails before write, registry unchanged (atomic write pattern) |
| T-03-07 | Denial of Service | generate_delta_files with 50+ patches | mitigate | Add warning log if >50 patches per FEAT; process in batches of 10 per change_class to avoid context exhaustion |
| T-03-08 | Information Disclosure | Settlement report paths | accept | Report written to feat_dir only; no external exposure; low-value target |
