# 027-02-PLAN: Enhance test_orchestrator.py with --verify-bugs support

## Goal
Add --verify-bugs targeted mode and --verify-mode=full-suite support to qa-test-run.

## Requirements
- VERIFY-01 (targeted mode)
- VERIFY-02 (full-suite mode)
- VERIFY-03 (post-verify transitions)

## Tasks

### Task 1: Add parameters to run_spec_test()
1. Add `verify_bugs: bool = False` parameter
2. Add `verify_mode: str = "targeted"` parameter ("targeted" | "full-suite")

### Task 2: Add test unit filtering
1. Implement `_filter_test_units_by_fixed_bugs()` helper
   - Get fixed bugs from bug registry
   - Extract coverage_ids from fixed bugs
   - Filter test_units to only include those coverage_ids
2. Apply filtering when `verify_bugs=True` and `verify_mode="targeted"`

### Task 3: Add post-execution bug state transitions
1. After test execution completes, if `verify_bugs=True`:
   - Get fixed bugs before execution
   - Determine which bugs passed/failed
   - Call `transition_after_verify()` on each fixed bug
   - Check auto-close conditions and close if met

## Success Criteria
- [ ] `--verify-bugs` targeted mode only runs tests for fixed bugs
- [ ] `--verify-mode=full-suite` runs all tests
- [ ] Post-execution state transitions work correctly
- [ ] Auto-close triggers when 2 conditions are met
