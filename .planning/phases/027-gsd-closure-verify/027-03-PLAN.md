# 027-03-PLAN: Add CLI commands and shadow fix detection

## Goal
Add ll-bug-transition, ll-bug-remediate, ll-bug-check-shadow commands, and shadow fix detection.

## Requirements
- CLI-01 (ll-bug-transition)
- CLI-02 (ll-bug-remediate)
- SHADOW-01 (shadow fix detection)

## Tasks

### Task 1: Add ll-bug-transition command to command.py
1. Add "bug-transition" to the allowed actions in _skill_handler()
2. Extract parameters: bug_id, to_state, reason, duplicate_of
3. Call transition_bug_status() with the parameters
4. Return success result

### Task 2: Add ll-bug-remediate command to command.py
1. Add "bug-remediate" to the allowed actions
2. Extract parameters: feat_ref, bug_id, batch
3. Load bug registry and show bug preview
4. Confirm with user before creating phase directory
5. Call bug_phase_generator.py

### Task 3: Add ll-bug-check-shadow command to command.py
1. Add "bug-check-shadow" to the allowed actions
2. Create shadow_detect.py module with:
   - scan_git_diff(workspace_root, since_commit)
   - check_shadow_fixes(workspace_root, feat_ref)
3. Run shadow detection and show warnings

### Task 4: Clean up gate_remediation.py
1. Remove duplicate _write_audit_log function
2. Simplify promote_detected_to_open() and archive_detected_not_in_gap_list()
   - Remove explicit audit log calls (now handled by transition_bug_status())

## Success Criteria
- [ ] ll-bug-transition works for all valid transitions
- [ ] ll-bug-remediate shows preview and creates phase directory
- [ ] ll-bug-check-shadow detects shadow fixes
- [ ] gate_remediation.py is cleaned up
