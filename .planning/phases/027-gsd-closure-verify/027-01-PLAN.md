# 027-01-PLAN: Enhance bug_registry.py with audit logging and verification helpers

## Goal
Add centralized audit logging to transition_bug_status(), verification helper functions, and fix_commit support.

## Requirements
- VERIFY-03 (post-verify transitions)
- AUDIT-01 (audit logging for all state changes)

## Tasks

### Task 1: Move audit logging to bug_registry.py
1. Move `_write_audit_log` from `gate_remediation.py` to `bug_registry.py`
2. Modify `transition_bug_status()` to automatically write audit log entries
3. Add optional `actor` and `run_id` parameters to `transition_bug_status()`
4. Update timestamps (`fixed_at`, `verified_at`, `closed_at`) on relevant state transitions

### Task 2: Add verification helper functions
1. `get_fixed_bugs(workspace_root, feat_ref)`: Get all bugs with status="fixed"
2. `get_bugs_by_coverage_ids(bugs, coverage_ids)`: Map coverage IDs to bug objects
3. `transition_after_verify(bug, passed, run_id)`: Handle fixed→re_verify_passed/open
4. `check_auto_close_conditions(bug, test_results_since_fix)`: Check 2 conditions for auto-close

### Task 3: Add fix_commit support
1. Allow setting `fix_commit` field when transitioning to "fixed" state
2. Add `set_fix_commit(bug, commit_hash)` helper function

## Success Criteria
- [ ] All state transitions automatically write to audit.log
- [ ] Helper functions have unit tests
- [ ] Timestamps are correctly updated
- [ ] fix_commit field can be set
