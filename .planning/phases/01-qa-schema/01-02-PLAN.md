**Validation functions** (each loads YAML, parses into dataclass, validates enums/required fields):
```python
def validate_plan(file_path: str) -> dict[str, list[str]]:
    """Validate a plan YAML file (api_test_plan or e2e_journey_plan).
    Returns {errors: [...], warnings: [...]}. Empty errors = valid.
    """
    # 1. Load YAML with yaml.safe_load()
    # 2. Check root key exists (api_test_plan or e2e_journey_plan)
    # 3. For api_test_plan: validate feature_id is non-empty string, source_feat_refs is non-empty list,
    #    api_objects has name+capabilities for each, priorities has p0/p1/p2 keys with list values
    # 4. For e2e_journey_plan: validate prototype_id, source_prototype_refs, journeys (journey_id, type in enum, priority in enum)
    # 5. Validate cut_records fields if present
    # 6. Return errors list (empty = valid)

def validate_manifest(file_path: str) -> dict[str, list[str]]:
    """Validate a manifest YAML file (api_coverage_manifest or e2e_coverage_manifest)."""
    # 1. Load YAML
    # 2. Check root key (api_coverage_manifest or e2e_coverage_manifest)
    # 3. For each item: validate required fields present (coverage_id, feature_id, capability, priority)
    # 4. Validate lifecycle_status in valid enum values
    # 5. Validate mapping_status in valid enum values
    # 6. Validate evidence_status in valid enum values
    # 7. Validate waiver_status in valid enum values
    # 8. Validate rerun_count >= 0
    # 9. Validate priority in ["P0", "P1", "P2"]
    # 10. Return errors list

def validate_spec(file_path: str) -> dict[str, list[str]]:
    """Validate a spec YAML file (api_test_spec or e2e_journey_spec)."""
    # 1. Load YAML
    # 2. Check root key (api_test_spec or e2e_journey_spec)
    # 3. For api_test_spec: validate case_id, coverage_id, endpoint, capability are non-empty strings
    #    request.body exists, expected.status_code is integer, expected.response_assertions is list
    #    evidence_required is non-empty list
    # 4. For e2e_journey_spec: validate case_id, coverage_id, journey_id, entry_point are non-empty strings
    #    user_steps is non-empty list, expected_ui_states is non-empty list
    #    evidence_required is non-empty list
    # 5. Return errors list

def validate_settlement(file_path: str) -> dict[str, list[str]]:
    """Validate a settlement YAML file (api_settlement_report, e2e_settlement_report, release_gate_input, or gate_rules)."""
    # 1. Load YAML
    # 2. Check root key matches expected type
    # 3. For api_settlement_report: validate total_capabilities >= 0, total_coverage_items >= 0
    #    lifecycle_counts has all 9 keys, acceptance_conclusion in enum
    # 4. For release_gate_input: validate api.status in enum, e2e.status in enum, final_decision in enum
    #    counts are non-negative integers
    # 5. For gate_rules: validate all boolean fields are boolean, fail_if_* fields are integers >= 0
    # 6. Return errors list
```

**CLI entry point** (at module bottom, guarded by `if __name__ == "__main__":`):
```python
import sys
if __name__ == "__main__":
    if len(sys.argv) < 3:
        print("Usage: python qa_schemas.py <plan|manifest|spec|settlement> <file.yaml>")
        sys.exit(1)
    asset_type = sys.argv[1]
    file_path = sys.argv[2]
    validators = {"plan": validate_plan, "manifest": validate_manifest, "spec": validate_spec, "settlement": validate_settlement}
    if asset_type not in validators:
        print(f"Unknown asset type: {asset_type}. Must be one of: plan, manifest, spec, settlement")
        sys.exit(1)
    errors = validators[asset_type](file_path)
    if errors:
        for e in errors:
            print(f"ERROR: {e}")
        sys.exit(1)
    else:
        print(f"OK: {file_path} is valid {asset_type}")
        sys.exit(0)
```

- Each validate function returns a dict with "errors" key (list of error strings) and "warnings" key (list of warning strings)
- Enum validation: check string values against hardcoded valid sets
- Required field validation: check key presence and non-None/non-empty
- Type validation: check int fields are int, list fields are list, dict fields are dict
- The validator is for API-side assets primarily (per CONTEXT.md decision: "API side first, E2E in Phase 2-3"), but should still validate E2E file structures if provided
