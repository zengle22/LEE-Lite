Part A — agents/executor.md (LLM prompt for spec-to-pytest generation):
```
skill: ll-qa-api-spec-to-tests
version: "1.0"

## Role
Convert frozen api-test-spec YAML into executable pytest test scripts with embedded evidence collection.

## Instructions
1. Read the api-test-spec YAML from {spec_path}
2. Validate it has all required fields: case_id, coverage_id, endpoint, capability, expected
3. Generate a pytest test file at {output_dir}/test_{case_id}.py containing:
   a. Import: pytest, yaml, pathlib
   b. EVIDENCE_DIR = Path("ssot/tests/.artifacts/evidence") with run_id-scoped subdirectory
   c. Test class: Test{Capability}Case with docstring showing case_id and coverage_id
   d. Test method: test_{capability}_{scenario} that:
      - Sets up preconditions from spec.preconditions
      - Builds request from spec.request (method, path_params, query_params, headers, body)
      - Executes request using requests library (or provided api_client fixture)
      - Asserts response.status_code == spec.expected.status_code
      - Iterates spec.expected.response_assertions and asserts each
      - Iterates spec.expected.side_effect_assertions and asserts each
      - Collects evidence as YAML dict with structure matching ADR-047 §6.3:
        evidence_record:
          case_id, coverage_id, executed_at (ISO timestamp), run_id
          evidence: { request_snapshot, response_snapshot, assertion_results }
          side_effects: [...]
          execution_status: "success" or "failed"
      - Writes evidence to EVIDENCE_DIR / {run_id} / {coverage_id}.evidence.yaml
   e. Handles errors gracefully: on exception, write evidence with execution_status: "error"
4. The generated script must NOT modify the spec file (specs are frozen)
5. The generated script must write evidence YAML for every item in spec.evidence_required

## Output Format
Write the generated .py file to {output_path}
```
