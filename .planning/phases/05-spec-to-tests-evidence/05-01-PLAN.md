Create cli/lib/qa_manifest_backfill.py with atomic manifest update functionality.

The module must export:
```python
def backfill_manifest(
    manifest_path: str | Path,
    results: list[dict],
    run_id: str,
) -> dict:
    """Atomically update manifest with execution results.

    Args:
        manifest_path: Path to api-coverage-manifest.yaml
        results: List of dicts with keys:
            - coverage_id: str
            - passed: bool
            - evidence_path: str (relative path to evidence YAML)
        run_id: Execution run identifier (e.g., "run-1234567890-12345")

    Returns:
        Dict with updated_count, run_id, manifest_path
    """
```

Implementation:
1. Read manifest YAML (use yaml.safe_load)
2. Validate manifest via validate_manifest() from qa_schemas
3. For each result, find matching manifest item by coverage_id
4. Update item fields:
   - lifecycle_status: "passed" if passed else "failed"
   - evidence_status: "complete"
   - evidence_refs: append evidence_path
   - last_run_id: run_id
   - rerun_count: increment by 1
5. Write to temp file first, then os.replace() for atomicity
6. Re-validate written manifest
7. Return summary dict

Follow immutability pattern: read full manifest -> build new dict -> write atomically.
Do NOT mutate in-place. Use pathlib.Path, yaml.safe_dump with sort_keys=False.
