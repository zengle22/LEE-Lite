Part A — agents/executor.md (LLM prompt for E2E spec-to-Playwright generation):
```
skill: ll-qa-e2e-spec-to-tests
version: "1.0"

## Role
Convert frozen e2e-journey-spec YAML into executable Playwright test scripts with embedded evidence collection.

## Instructions
1. Read the e2e-journey-spec YAML from {spec_path}
2. Validate it has all required fields: case_id, coverage_id, journey_id, entry_point, user_steps
3. Generate a Playwright test file at {output_dir}/{case_id}.spec.ts containing:
   a. Import: { test, expect } from '@playwright/test'
   b. Test using test() function with descriptive name from case_id
   c. Test body that:
      - Navigates to entry_point via page.goto()
      - Iterates user_steps: performs action (click, fill, navigate) on target selector/locator
      - Asserts expected_ui_states: expect(page.locator(selector)).toHaveText/toBeVisible
      - Asserts expected_persistence: reload page via page.reload() and verify state persists
      - Checks anti_false_pass_checks: no console errors (listen to page.on('console')), no 404 responses
      - Collects evidence per ADR-047 §6.3 E2E evidence format:
        - Screenshot via page.screenshot()
        - Network events intercepted via page.on('response')
        - DOM assertion results recorded as objects { selector, expected, result }
      - Writes evidence to {evidence_dir}/{run_id}/{coverage_id}.evidence.yaml
        using Node.js fs.writeFileSync with YAML string format
   d. Handles errors: on exception, record failure and write evidence with execution_status: "error"
4. The generated script must NOT modify the spec file (specs are frozen)
5. The generated script must write evidence YAML for every item in spec.evidence_required
6. Output is a TypeScript .spec.ts file, not Python

## Output Format
Write the generated .spec.ts file to {output_path}
```

Part B — agents/supervisor.md (validation checklist):
```
skill: ll-qa-e2e-spec-to-tests
version: "1.0"

## Supervisor Validation Checklist
1. Generated .spec.ts file exists at expected output path
2. File contains import from '@playwright/test'
3. File contains test() with correct case_id in name/description
4. File iterates user_steps from spec (page.click, page.fill, or page.goto calls)
5. File asserts expected_ui_states (expect(locator).toHaveText or toBeVisible)
6. File asserts expected_persistence (page.reload() followed by assertion)
7. File collects E2E evidence (page.screenshot, page.on('response'), console listener)
8. File writes evidence YAML for every spec.evidence_required item
9. File does NOT modify the input spec file
10. File has try/catch error handling with evidence write in catch block
11. Evidence path includes run_id for collision avoidance
```
