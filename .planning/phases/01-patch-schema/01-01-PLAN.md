```yaml
# ADR-049 QA Schema: Experience Patch
# Source: ADR-049 §5.3, §5.1
# Scope: defines the structure of individual experience patch YAML files
# Must NOT contain: execution logic, script content

experience_patch:
  # Required top-level fields
  id: string              # UXPATCH-0001 format (note: uses hyphen UXPATCH-0001 not underscore)
  type: string            # fixed value: "experience_patch"
  status: enum[draft, active, validated, pending_backwrite, backwritten, retain_in_code, upgraded_to_src, superseded, discarded, archived]
  created_at: datetime    # ISO 8601
  updated_at: datetime    # ISO 8601
  title: string
  summary: string
  source: object          # see source sub-fields below
  scope: object           # see scope sub-fields below
  change_class: enum[visual, interaction, semantic]

  # Optional top-level fields
  severity: enum[low, medium, high]
  conflict: boolean       # default: false
  conflict_details: object
  problem: object         # user_issue, evidence
  decision: object        # code_hotfix_allowed, must_backwrite_ssot, backwrite_targets, backwrite_deadline
  implementation: object  # code_changed, changed_files
  test_impact: object     # see test_impact sub-fields below
  related_ids: array[string]
  resolution: object      # see resolution sub-fields below

  # source sub-fields (required object)
  source:
    from: string          # fixed: "product_experience"
    actor: enum[human, ai_suggested]
    session: string
    prompt_ref: string
    ai_suggested_class: enum[visual, interaction, semantic]  # optional
    human_confirmed_class: enum[visual, interaction, semantic]  # required

  # scope sub-fields (required object)
  scope:
    feat_ref: string
    ui_ref: string        # optional
    tech_ref: string      # optional
    page: string
    module: string

  # test_impact sub-fields (optional object)
  test_impact:
    impacts_user_path: boolean
    impacts_acceptance: boolean
    impacts_existing_testcases: boolean
    affected_routes: array[string]
    test_targets: array[string]

  # resolution sub-fields (optional object)
  resolution:
    backwrite_status: enum[pending, backwritten, discarded, upgraded_to_src, superseded]
    merged_into_ssot_at: datetime | null
    src_created: string | null  # SRC ID
    closed_at: datetime | null
```

Key conventions from existing schemas:
- Header comments identify ADR source, scope, and what it must NOT contain
- Use `enum[...]` for enumeration types
- Use `| null` for nullable fields
- Use `array[string]` for string arrays
- Use `datetime` for timestamp fields
- Group sub-fields under their parent object with indentation
