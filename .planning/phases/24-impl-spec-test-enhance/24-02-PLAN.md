1. Remove these constants entirely (per D-05):
   - `STRONG_API_KEYWORDS`
   - `WEAK_API_KEYWORDS`
   - `NEGATION_MARKERS`

2. Add `API_SURFACE_PATTERNS` constant to replace them:
   ```python
   API_SURFACE_PATTERNS = [
       # REST/gRPC endpoint patterns
       r"/v\d+/", r"\b(GET|POST|PUT|PATCH|DELETE)\s+/",
       # Command handler patterns
       r"command\s+handler", r"handler\s+contract",
       # Backend service interface patterns
       r"backend\s+service", r"service\s+interface",
       # Webhook patterns
       r"webhook", r"callback\s+contract",
       # API/schema patterns
       r"\bapi\b.*\b(contract|endpoint|schema)\b", r"\bREST\b", r"\bgRPC\b",
   ]
   ```

3. Add `detect_api_surface_in_scope(feature: dict[str, Any]) -> bool` function:
   - Collect text from `feature.get("scope")` and `feature.get("outputs")` only (per D-06: primary signal is scope/outputs).
   - Flatten to strings, join with "\n", lowercase.
   - Return True if any `API_SURFACE_PATTERNS` regex matches the text.
   - For engineering baseline FEATs (detected via `is_engineering_baseline_feature()`), return False unless an explicit API surface pattern is found (per D-08: conservative for engineering baseline).

4. Modify `assess_optional_artifacts()`:
   - Remove `arch_hits = keyword_hits(feature, ARCH_KEYWORDS)` — keep ARCH keyword detection as-is (only API detection changes per D-05).
   - Remove `api_hits = keyword_hits(feature, STRONG_API_KEYWORDS)` and `weak_api_hits = keyword_hits(feature, WEAK_API_KEYWORDS)`.
   - Replace with `api_required = detect_api_surface_in_scope(feature)`.
   - For the `is_engineering` branch: `api_required = detect_api_surface_in_scope(feature)` (no special weak_api logic).
   - For the non-engineering branch: `api_required = detect_api_surface_in_scope(feature)` (no axis-based API inference).
   - Update `api_rationale`: if api_required, use `["API required by explicit surface declaration in FEAT scope/outputs."]`; else use `["API omitted because FEAT scope/outputs do not declare an explicit command-level contract surface."]`.
   - Remove `api_hits` and `weak_api_hits` from the returned dict (or set to empty list).

5. Keep `keyword_hits()` function for potential future use but it no longer needs NEGATION_MARKERS.
