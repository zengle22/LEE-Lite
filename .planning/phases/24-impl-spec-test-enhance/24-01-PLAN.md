  <action>
Add the following to `impl_spec_test_skill_guard.py` after the imports and before `validate_input()`:

1. Define `_HEADING_RE = re.compile(r'^(#{2,4})\s+(.+)$', re.MULTILINE)` — matches level 2-4 headings with 1+ spaces after hashes.

2. Define `_MAX_EXCERPT_LINES = 3` constant.

3. Define `_parse_markdown_sections(content: str) -> list[dict[str, Any]]`:
   - Split content into lines, enumerate from 1.
   - For each line matching `_HEADING_RE`:
     - `heading_level = len(match.group(1))`
     - `heading_text = match.group(2).strip()`
     - Collect up to `_MAX_EXCERPT_LINES` non-empty lines immediately following the heading line (skip the heading line itself).
     - `excerpt = "\n".join(collected_lines) if collected_lines else heading_text`
     - Append dict: `{"heading_level": heading_level, "heading_text": heading_text, "excerpt": excerpt, "line_number": line_number}`
   - Return the list.

4. Define `validate_markdown_sections(path: Path) -> list[dict[str, str]]`:
   - Read file as utf-8 text.
   - Call `_parse_markdown_sections(content)`.
   - If no sections found, return `[{"severity": "warning", "title": "No markdown sections detected", "detail": f"File {path.name} contains no recognizable headings."}]`
   - Otherwise return empty list (no defects from parsing itself).

5. Add `import re` at top of file if not present.

Per D-01 (hybrid parsing): regex is primary. No AST parser needed unless complex nesting detected — for this phase, regex-only is sufficient per Claude's discretion in CONTEXT.md.
Per D-02: The parser works on any markdown document, not just specific files.
Per D-03: Regex matches both Chinese and mixed headings via `.+` capture.
Per D-04: Excerpt is 2-3 lines (up to `_MAX_EXCERPT_LINES`) with fallback to heading text.
  </action>
