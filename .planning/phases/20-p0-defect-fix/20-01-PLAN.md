   - `API_DUPLICATE = "api_authority_duplicate"`

2. Define `@dataclass(frozen=True) class Violation`:
   - `type: ViolationType`
   - `severity: str` — `"blocker"` or `"warning"`
   - `file_path: str`
   - `anchor_id: str | None = None`
   - `detail: str = ""`
   - `evidence: dict[str, Any] = field(default_factory=dict)`

3. Define `@dataclass(frozen=True) class ScanResult`:
   - `total_violations: int`
   - `blocker_count: int`
   - `warning_count: int`
   - `violations: list[Violation]`

4. Define `OVERLAY_TERMS = frozenset([...])` with:
   - "governance", "gate", "handoff", "formal", "registry",
   - "validation", "audit", "bypass", "settlement"

5. Implement `scan_overlay_elevation(ssot_dir: Path) -> list[Violation]`:
   - Scan `ssot/epic/*.md` and `ssot/feat/*.md`
   - Read file content (simple text scan, no heavy parsing)
   - Look for primary object indicators (looking for patterns like "## Primary Object" or the first heading content)
   - Heuristic: if any overlay term appears prominently near the start of the document as a main topic
   - Return `Violation(type=OVERLAY_ELEVATION, severity="blocker", ...)` for each match

6. Implement `scan_api_duplicates(ssot_dir: Path) -> list[Violation]`:
   - Scan `ssot/api/*.md` and `ssot/api/*.yaml`
   - Extract API endpoints/paths (look for patterns like "GET /api/..." or "endpoint: /api/...")
   - Build a map: `endpoint -> list[file_path]`
   - Return `Violation(type=API_DUPLICATE, severity="blocker", ...)` for each endpoint with >1 file

7. Implement `scan_ssot(ssot_dir: Path) -> ScanResult`:
   - Call `scan_overlay_elevation()` and `scan_api_duplicates()`
   - Combine violations
   - Count totals and blockers
   - Return `ScanResult(...)`

8. All functions use `CommandError` + `ensure()` for preconditions.
9. All function signatures have type annotations. Use `from __future__ import annotations`.
10. Import from `cli.lib.errors`, `dataclasses`, `enum`, `pathlib`, `typing`, `re` (for endpoint extraction).
