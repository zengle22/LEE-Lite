   - Generated by ll-qa-api-manifest-init

3. ssot/tests/api/{feat_id}/api-test-spec/SPEC-*.md
   - Generated by ll-qa-api-spec-gen

### agents/orchestrator.md
Write AI agent orchestrator that:
1. Validates feat_ref and feat_freeze_package status=frozen
2. Calls Skill tool: `/ll-qa-feat-to-apiplan` with feat_freeze_package + feat_ref
3. On success: validates Acceptance Traceability table completeness (each capability has AC mapping, each AC has capability coverage)
4. If validation fails: abort and report missing mappings
5. On validation pass: calls Skill tool: `/ll-qa-api-manifest-init` with api-test-plan output path
6. On manifest-init success: calls Skill tool: `/ll-qa-api-spec-gen` with api-coverage-manifest output path
7. On spec-gen failure: log warning but preserve manifest (continue strategy)
8. Returns execution summary with paths to all generated artifacts

State machine states: START, APIPLAN_RUNNING, TRACEABILITY_VALIDATING, MANIFEST_RUNNING, SPEC_RUNNING, COMPLETE, FAILED

3. api-coverage-manifest.yaml was generated
4. At least one api-test-spec/*.md was generated
5. All artifacts are at canonical paths under ssot/tests/api/{feat_id}/
