### Step 4: update_manifest() with optimistic locking
```python
def update_manifest(
    workspace_root: Path,
    manifest_items: list[dict[str, Any]],
    run_id: str,
) -> None:
    """Update coverage manifest with execution results.

    Uses timestamp + version optimistic lock (Windows compatible).
    """
    import yaml
    from uuid import uuid4
    from datetime import datetime

    # Determine manifest path based on chain
    if manifest_items and "feat_ref" in manifest_items[0]:
        manifest_path = workspace_root / f"ssot/tests/api/{manifest_items[0]['feat_ref']}/api-coverage-manifest.yaml"
    else:
        manifest_path = workspace_root / f"ssot/tests/e2e/{manifest_items[0]['proto_ref']}/e2e-coverage-manifest.yaml"

    manifest = yaml.safe_load(manifest_path.read_text(encoding="utf-8")) or {}

    # Optimistic lock: read version, update, write with new version
    expected_version = manifest.get("_version", "0")
    for item in manifest_items:
        for existing in manifest.get("items", []):
            if existing["coverage_id"] == item["coverage_id"]:
                existing["lifecycle_status"] = item.get("status", "executed")
                existing["last_run_id"] = run_id
                existing["last_run_status"] = item.get("run_status", "passed")
                if item.get("evidence_ref"):
                    existing.setdefault("evidence_refs", []).append(item["evidence_ref"])

    manifest["_version"] = str(uuid4())
    manifest["_last_updated"] = datetime.utcnow().isoformat() + "Z"

    manifest_path.write_text(yaml.safe_dump(manifest, allow_unicode=True), encoding="utf-8")
```

# LL QA Test Run

User-facing CLI skill for running QA test chains end-to-end.

## Input (CLI args)
- --feat-ref: feat identifier (API chain)
- --proto-ref: prototype identifier (E2E chain)
- --app-url: frontend application URL (E2E chain required, default http://localhost:3000)
- --api-url: backend API URL (separated architecture)
- --chain: execution chain type (api | e2e | both)
- --coverage-mode: smoke | qualification
- --resume: resume from last failed run
- --resume-from: resume from specific run_id

## Output
- Updated api-coverage-manifest.yaml or e2e-coverage-manifest.yaml
- Execution evidence at artifacts/active/qa/candidates/

## Internal Flow
1. provision_environment() → ENV file
2. spec_to_testset() → SPEC_ADAPTER_COMPAT file
3. execute_test_exec_skill() → test execution
4. update_manifest() → manifest update with optimistic lock

## Options
- --resume: re-run only failed coverage items from last run
