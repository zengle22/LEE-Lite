Create cli/lib/silent_override.py with the following structure:

1. **OverrideResult dataclass** (frozen):
```python
@dataclass(frozen=True)
class OverrideResult:
    passed: bool
    classification: str  # "ok" | "clarification" | "semantic_change"
    semantic_drift: dict  # D-06: must include has_drift, drift_results, classification
    block_reasons: list[str]
    pass_with_revisions: list[str]
```

2. **_extract_frz_ref(output_data, frz_id)** function:
   - First check if frz_id is provided as parameter
   - If not, try to extract frz_ref from output_data dict (look for "frz_ref" key)
   - If neither found, raise CommandError("INVALID_REQUEST", "No FRZ reference found in output or parameters")

3. **_load_frz_package(workspace_root, frz_id)** function:
   - Use frz_registry.get_frz() to look up FRZ record
   - Validate record exists and status == "frozen"
   - Load FRZ YAML from record["package_ref"]
   - Parse with frz_schema._parse_frz_dict()
   - Return FRZPackage

4. **classify_change(drift_results, disallowed_fields)** function (D-08, D-09):
   - Input: list of DriftResult + list of disallowed field names
   - Rule-based classification (no LLM):
     - If any drift_type == "missing" → return "semantic_change"
     - If any drift_type == "tampered" → return "semantic_change"
     - If any drift_type == "constraint_violation" → return "semantic_change"
     - If disallowed_fields is non-empty → return "semantic_change"
     - If any drift_type == "new_field" but field is within derived_allowed → return "clarification"
     - If all drift_type == "none" → return "ok"
   - This implements D-09: anchor name matches + supplementary content = clarification; anchor name differs or contradicts = semantic_change

5. **check_silent_override(frz_package, output_data, anchor_filter=None)** function:
   - Extract all anchor IDs from frz_package (JRN-xxx from core_journeys, ENT-xxx from domain_model, SM-xxx from state_machine, UNK-xxx from known_unknowns)
   - If anchor_filter is set (for layered baselines D-07), filter anchors to only those matching the filter pattern
   - Build target_data dict keyed by anchor_id from output_data
   - For each anchor_id, call drift_detector.check_drift(anchor_id, frz_package, target_data)
   - Call drift_detector.check_derived_allowed(frz_package, output_data) for field-level checks
   - Call drift_detector.check_constraints(frz_package, output_data) for constraint checks
   - Call drift_detector.check_known_unknowns(frz_package, output_data) for expired KU checks
   - Apply D-04 block rules:
     - anchor missing → add to block_reasons
     - tampered → add to block_reasons
     - constraint_violation → add to block_reasons
     - new_field outside derived_allowed → add to block_reasons
   - Apply D-05 pass_with_revisions rules:
     - extra fields within derived_allowed → add to pass_with_revisions
     - expired known_unknown still open → add to pass_with_revisions
   - Call classify_change() to get classification
   - Return OverrideResult with:
     - passed = len(block_reasons) == 0
     - classification from classify_change()
     - semantic_drift dict (D-06): {"has_drift": any_drift, "drift_results": [dr.to_dict() for dr in drift_results], "classification": classification}
     - block_reasons list
     - pass_with_revisions list

6. **CLI entry point** (main function with argparse):
   - `python cli/lib/silent_override.py check --output <artifacts-dir> --frz <frz-id> [--workspace <workspace-root>] [--mode full|journey_sm|product_boundary]`
   - Loads output data from artifacts-dir (read JSON/YAML files)
   - Runs check_silent_override()
   - Prints result as JSON to stdout
   - Exit code 0 if passed, 1 if blocked

7. **Layered baseline anchor_filter mapping** (for D-07, used by 09-03):
   - mode="full": anchor_filter = None (all anchors)
   - mode="journey_sm": anchor_filter = {"JRN", "SM"} (only journey and state machine anchors)
   - mode="product_boundary": anchor_filter = set() (skip anchor checks, only check product_boundary + constraints)

Import order: stdlib first, then project libs (drift_detector, frz_schema, frz_registry, errors, anchor_registry).
Use frozen dataclasses for all result types. Use ensure() for input validation.

| Boundary | Description |
|----------|-------------|
| FRZ registry → silent_override.py | FRZ records could be tampered; must validate status == "frozen" |
| Output artifacts → silent_override.py | Skill output could inject fake clean results; drift computed independently |
| CLI parameters → silent_override.py | FRZ ID parameter could reference non-existent FRZ; must validate existence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | _load_frz_package | mitigate | Validate FRZ record exists via frz_registry.get_frz and status == "frozen" before loading; CommandError on invalid FRZ |
| T-09-02 | Tampering | check_silent_override | mitigate | Compute drift independently using drift_detector functions; never trust upstream drift results passed in output_data |
| T-09-03 | Tampering | classify_change | mitigate | Rule-based classification using DriftResult drift_type values computed locally; no LLM or external judgment |
| T-09-04 | Spoofing | CLI --frz parameter | mitigate | Validate FRZ ID matches FRZ_ID_PATTERN regex; verify package_ref path exists on filesystem |
| T-09-05 | Information Disclosure | OverrideResult JSON output | accept | Output contains only verdict and drift metadata; no sensitive user data or internal system paths exposed |
