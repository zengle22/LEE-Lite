**File 2: `cli/lib/task_pack_schema.py`** — Python validation module.

Model after `frz_schema.py` and `patch_schema.py`. Use this exact structure:

1. Imports: `from __future__ import annotations`, `dataclass`, `field` from dataclasses, `Enum` from enum, `Path` from pathlib, `Any` from typing, `yaml`, `re`

2. Enums (all `str, Enum`):
   - `TaskStatus`: pending, running, passed, failed, still_failed, skipped, blocked
   - `TaskType`: impl, test_api, test_e2e, review, doc, gate

3. Frozen dataclasses:
   - `@dataclass(frozen=True) class Task`:
     - `task_id: str`
     - `type: TaskType`
     - `title: str`
     - `depends_on: list[str] = field(default_factory=list)`
     - `status: TaskStatus = TaskStatus.pending`
     - `verifies: list[str] = field(default_factory=list)`
   - `@dataclass(frozen=True) class TaskPack`:
     - `artifact_type: str = "task_pack"`
     - `pack_id: str | None = None`
     - `feat_ref: str | None = None`
     - `created_at: str | None = None`
     - `tasks: list[Task] = field(default_factory=list)`

4. Custom error class: `class TaskPackSchemaError(ValueError)`

5. Helper functions:
   - `_require(data: dict, key: str, label: str) -> None` — raises TaskPackSchemaError if key missing or None
   - `_enum_check(value: str, enum_cls: type[Enum], label: str, field_name: str) -> None` — raises TaskPackSchemaError if value not in enum
   - `_parse_task_dict(data: dict) -> Task` — converts raw YAML dict to Task instance, with enum conversion and default handling for depends_on/verifies (use `or []` pattern)
   - `_parse_task_pack_dict(data: dict) -> TaskPack` — converts raw YAML dict to TaskPack instance, iterates tasks through _parse_task_dict

6. `validate(data: dict) -> TaskPack` — validates a raw YAML dict:
   - Extract `task_pack` key if present, else use data directly
   - Call `_require` for: pack_id, feat_ref, created_at, tasks
   - Validate tasks is a non-empty list
   - Validate all task_ids are unique (raise TaskPackSchemaError if duplicates)
   - Validate each task has required fields: task_id, type, title, status
   - Validate type and status via `_enum_check`
   - Validate no orphan depends_on refs (each dep in depends_on must reference an existing task_id in the same pack)
   - Parse and return TaskPack via `_parse_task_pack_dict`

7. `_TASK_PACK_SCHEMA_DIR = Path(__file__).parent.parent.parent / "ssot" / "schemas" / "qa"`

8. `_load_yaml(path: Path) -> dict` — uses yaml.safe_load (never yaml.load)

9. `validate_file(path: str | Path) -> TaskPack` — loads YAML file, calls validate

10. `_VALIDATORS = {"task_pack": ("task_pack", _parse_task_pack_dict)}`

11. `_detect_schema_type(data: dict) -> str | None` — checks for "task_pack" top-level key

12. `main()` — CLI entry point, model after `frz_schema.py` main()

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML file → validator | Untrusted YAML files parsed and validated — must use safe_load |
| Schema file → parser | Human-written schema YAML parsed for reference — read-only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | YAML parsing (`yaml.safe_load`) | mitigate | Use `yaml.safe_load()` exclusively — never `yaml.load()` — prevents arbitrary code execution via YAML deserialization (per frz_schema.py pattern) |
| T-11-02 | Information Disclosure | Error messages from `validate()` | accept | Schema validation errors may reveal internal field names; low risk since this is a local dev tool, not a network service |
| T-11-03 | Tampering | `validate_file()` path argument | mitigate | Path traversal not applicable — files are read from `ssot/schemas/qa/` or user-specified paths in a local dev context; no network exposure |
</threat_model>
