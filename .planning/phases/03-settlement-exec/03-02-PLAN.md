## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 5 input YAML files → validate_input.sh | External YAML files cross trust boundary; must be validated before processing |
| LLM-generated gate decision → validate_output.sh | LLM output must conform to gate evaluation contract with correct enum values |
| Evidence files → SHA-256 hash computation | Evidence file contents used for hash; must be read in deterministic order |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | validate_input.sh manifest/settlement parsing | mitigate | Use `python -m cli.lib.qa_schemas --type manifest/settlement` which uses yaml.safe_load() only, never yaml.load() with unsafe Loader |
| T-03-06 | Tampering | validate_output.sh gate output parsing | mitigate | Inline Python uses yaml.safe_load() to parse output; validates final_decision enum, evidence_hash format |
| T-03-07 | Tampering | Evidence hash computation in gate evaluation | mitigate | Executor prompt specifies: sort evidence file paths alphabetically, read bytes, concatenate, compute SHA-256 — prevents non-deterministic ordering |
| T-03-08 | Repudiation | Gate decision generation | mitigate | release_gate_input.yaml includes evaluated_at timestamp, decision_reason, and all 7 anti-laziness check results as audit trail |
| T-03-09 | Elevation of Privilege | run.sh argument parsing | mitigate | Validate all 5 input paths are files that exist before processing; use set -euo pipefail to fail fast on undefined variables |
