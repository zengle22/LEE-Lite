   - Filters by `changed_files` matching `target_file` (path contains or fuzzy match)
   - Optionally filters by `feat_ref`
   - Returns list of patch dicts sorted by relevance

2. **`summarize_patch_for_context(patch: dict, max_tokens: int = 3000) -> str`**:
   - Formats patch as context-friendly text: ID, title, change_class, scope, description
   - Respects ADR-049 §12.1 context budget (3000 tokens max, 10 patches max)
   - For patches beyond 10th: show one-line summary only (ID + title + change_class)

3. **`inject_context(workspace_root: Path, target_files: list[str]) -> str`**:
   - Convenience function: for each target file, find related patches, format as context
   - Returns formatted context string ready for AI consumption
   - Output format: Markdown block with patch summaries

Do NOT reimplement patch scanning — use resolve_patch_context() from test_exec_artifacts.py.

  <action>Create `cli/lib/patch_auto_register.py` with:

1. **`detect_changes(workspace_root: Path, feat_ref: str | None = None) -> list[dict]`**:
   - Uses `git diff --name-status` to detect uncommitted changes
   - Maps changed files to FEAT directories and potential scopes
   - Returns list of change records: file_path, change_type, affected_scope

2. **`draft_patch_yaml(changes: list[dict], workspace_root: Path) -> dict`**:
   - Pre-fills Patch YAML structure:
     - `id`: auto-generated (UXPATCH-{timestamp})
     - `title`: auto-generated from changed files
     - `change_class`: suggested based on file patterns (.css→visual, .tsx→interaction, business logic→semantic)
     - `scope`: extracted from file paths (page, module)
     - `changed_files`: list from git diff
     - `test_impact`: bool (required per ADR-049 §10.1)
     - `backwrite_targets`: suggested based on change_class
     - `status`: draft
   - Returns dict ready for user review

3. **`register_patch(patch: dict, output_dir: Path) -> Path`**:
   - Validates required fields (id, change_class, scope, changed_files, test_impact)
   - Writes YAML to `ssot/experience-patches/{patch_id}.yaml`
   - Returns output path

Do NOT auto-commit patches — ADR-049 §12.2 requires user confirmation before入库.
  </action>

    - Does NOT auto-commit or auto-finalize patches
    - Uses git diff for change detection
