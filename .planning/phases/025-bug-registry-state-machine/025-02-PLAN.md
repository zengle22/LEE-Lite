      Task 1: Root Cause Analysis (type: auto) — analyze bug evidence from context, output fix_hypothesis
      Task 2: Implement Fix (type: auto) — minimal-scope fix based on root cause
      Task 3: Update Bug Status (type: auto) — transition fixing -> fixed
      Task 4: Verify Fix (type: auto) — run targeted test, transition fixed -> re_verify_passed or back to open

    Functions:
    - `_build_context_md(bug: dict[str, Any]) -> str` — renders CONTEXT.md from bug record fields: bug_id, case_id, title, status, severity, gap_type, actual, expected, diagnostics (bulleted list), evidence_ref, run_id, discovered_at
    - `_build_plan_md(bugs: list[dict[str, Any]], is_batch: bool) -> str` — renders PLAN.md with frontmatter (autonomous: false, phase name) + 6-task template. For batch: repeats task group per bug with bug_id prefix on each task name.
    - `_build_discussion_log_md() -> str` — returns "# Discussion Log\n\n" (empty placeholder)
    - `_build_summary_md() -> str` — returns "# Summary\n\n" (empty placeholder)
    - `generate_bug_phase(workspace_root: Path, bug: dict[str, Any], phase_number: int) -> Path` — single bug mode:
      1. Create dir: workspace_root / ".planning" / "phases" / f"{phase_number:03d}-bug-fix-{bug['bug_id']}"
      2. Write CONTEXT.md, PLAN.md (single bug), DISCUSSION-LOG.md, SUMMARY.md
      3. Return created directory Path
    - `generate_batch_phase(workspace_root: Path, bugs: list[dict[str, Any]], phase_number: int) -> Path` — batch mode (max 3 bugs):
      1. Validate len(bugs) <= 3, raise CommandError if exceeded
      2. Hash bug_ids for batch dir name: hashlib.md5("-".join(b["bug_id"] for b in bugs).encode()).hexdigest()[:8]
      3. Create dir: workspace_root / ".planning" / "phases" / f"{phase_number:03d}-bug-fix-batch-{hash}"
      4. Write all 4 files, PLAN.md uses batch template (all bugs in one file)
      5. Return created directory Path

    All PLAN.md output must have frontmatter: `autonomous: false` (locked D-15).
