## 4. v2.2.1 关键修复详情

### 4.1 FIX-P1-09: impl-spec-test 中文解析问题

**问题**
- impl-spec-test 当前只验证 JSON 信封，不能解析带有中文章节的 markdown 文档
- 无法正确提取片段，错误报告不完整

**解决方案**
- 新增 `_parse_markdown_sections()` 函数，支持中文标题解析
- 新增 `validate_markdown_sections()` 函数，验证并解析 markdown 文档
- 新增 8 个测试：
  - 带数字中文标题解析（如 "### 5.5 完成状态定义"）
  - 纯中文标题解析
  - 中英文混合标题解析
  - 片段提取限制最多 3 行
  - 片段回退到标题文本（无内容时）
  - 多个标题保持顺序
  - Level 1 标题被忽略
  - 无效标题（无空格）被忽略

**验证结果**
- ✅ 所有 17 个测试通过
- ✅ 中文 markdown 解析功能正常

---

### 4.2 ENH-P1-01: api_required 基于能力边界

**问题**
- 之前基于关键词匹配，不够准确

**解决方案**
- 完全移除 `STRONG_API_KEYWORDS`、`WEAK_API_KEYWORDS`、`NEGATION_MARKERS`
- 新增 `API_SURFACE_PATTERNS`，基于正则表达式
- 新增 `detect_api_surface_in_scope()`，只检查 FEAT scope 和 outputs
- 对工程基线 FEAT 保持保守

---

### 4.3 ENH-P1-02: ssot_type 强制输出

**解决方案**
- TECH 文档 frontmatter 加 `ssot_type: "TECH"`
- ARCH 文档 frontmatter 加 `ssot_type: "ARCH"`
- API 文档 frontmatter 加 `ssot_type: "API"`
- JSON payload 的 optional block 也加 ssot_type

---

### 4.4 ENH-P1-03: API 前置条件与后置输出章节

**解决方案**
- 扩展 `DEFAULT_API_COMMAND_SPECS` 新增 7 个字段
- 新增 "前置条件与后置输出章节
- 状态转换表（6 列）
- 叙事总结
- 一致性检查验证

---

### 4.5 ENH-P1-04: source_refs 完整追溯

**解决方案**
- `build_refs()` 返回 epic_ref 和 src_ref
- `_normalized_source_refs()` 包含完整追溯链
- 类型标记格式：`ARCH:xxx`、`API:xxx`、`EPIC:xxx`、`SRC:xxx`、`SURFACE:xxx`

---

### 4.6 ENH-P1-05: TESTSET 自动触发

**解决方案**
- 新增 `_trigger_testset_generation()` 函数
- 在 `run_workflow()` 完成后触发
- 条件触发：api_required 或 frontend_required 为 true 时
- 非阻塞、即发即弃
- 写入审计日志到 testset-trigger-record.json
