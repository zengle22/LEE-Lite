# Review Checklist

- Does the skill keep the standard governed skill shell and contract split?
- Does the runtime create the expected LL project scaffold instead of a one-off custom layout?
- Are root config files and runtime shell files generated deterministically?
- Does the package record created, updated, and skipped paths clearly?
- Does the runtime refuse to overwrite unmanaged existing files?
- Do output validation and readiness validation check both package artifacts and required root paths?
