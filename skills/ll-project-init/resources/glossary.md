# Glossary

- managed file: a scaffold file owned by this skill and safe to create or refresh according to the write policy.
- runtime shell: a directory that exists as part of the repository topology but should not store durable governed truth.
- durable tree: the versionable project structure that stores specs, skills, CLI code, docs, artifacts, and SSOT objects.
- structural validation: deterministic checks such as schema shape, required file presence, and required path creation.
- semantic validation: review of layout fidelity, scope control, and no-runtime-residue rules.
- execution evidence: executor record of commands, file actions, decisions, and uncertainties.
- supervision evidence: supervisor record of semantic findings and the pass or revise decision.
