# Input Semantic Checklist

Review the selected FEAT for UI-spec derivation readiness.

- Is the selected FEAT really user-visible, or is it a backend-only capability that should not be forced through UI derivation?
- Are goal, trigger scenario, constraints, and success criteria explicit enough to derive a user path?
- Does the FEAT freeze product behavior rather than code structure?
- If UI hints are present, do they refine the FEAT instead of silently overriding it?
- Would downstream UI/TECH authors still need to guess core page purpose, user path, or state boundary after reading the FEAT?
