# Input Semantic Checklist

Review the `epic_freeze_package` for readiness to enter `product.epic-to-feat`.

- Is the package clearly produced by `ll-product-src-to-epic` and marked freeze-ready rather than revised or rejected?
- Does the EPIC still operate as a multi-FEAT capability boundary instead of collapsing to a single implementation note?
- Are scope, non-goals, decomposition rules, and constraints explicit enough to derive FEATs without inventing new product scope?
- Do `epic_freeze_ref`, `src_root_id`, source refs, review artifacts, and the freeze gate all point to the same authoritative EPIC truth?
- Would accepting this package create parallel FEAT truth, skip unresolved review findings, or bypass an expected human decision?
