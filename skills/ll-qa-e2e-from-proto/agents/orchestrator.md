# Orchestrator Agent: ll-qa-e2e-from-proto

## Role

Coordinate the complete E2E test chain from a prototype/FEAT to e2e-journey-plan + manifest + specs by sequencing sub-skills via the Skill tool.

## State Machine

```
START → E2EPLAN_RUNNING → TRACEABILITY_VALIDATING → MANIFEST_RUNNING → SPEC_RUNNING → COMPLETE
```

## Instructions

### Phase 1: START

1. Validate that either `prototype_freeze_package` OR `feat_ref` is provided
2. If prototype mode: verify prototype_freeze_package exists and is frozen
3. If api-derived mode: verify feat_ref exists in ll-product-epic-to-feat output
4. Transition to `E2EPLAN_RUNNING` state

### Phase 2: E2EPLAN_RUNNING

1. Call Skill tool `/ll-qa-prototype-to-e2eplan` with:
   - `prototype_freeze_package`: path to freeze package (proto mode)
   - OR `feat_ref`: the specific FEAT identifier (api-derived mode)
2. On failure: abort and report error with stage `E2EPLAN_RUNNING`
3. On success: extract output path (typically `ssot/tests/e2e/{proto_id}/e2e-journey-plan.md`)
4. Transition to `TRACEABILITY_VALIDATING` state

### Phase 3: TRACEABILITY_VALIDATING

1. Read the generated `e2e-journey-plan.md`
2. Validate Acceptance Traceability table is present:
   - Each journey must have at least one `acceptance_ref` mapping
   - Each `acceptance_ref` must have at least one journey coverage
3. If validation fails: abort and report missing mappings with stage `TRACEABILITY_VALIDATING`
4. On validation pass: transition to `MANIFEST_RUNNING` state

### Phase 4: MANIFEST_RUNNING

1. Call Skill tool `/ll-qa-e2e-manifest-init` with:
   - `e2e_journey_plan_path`: path to the e2e-journey-plan.md from Phase 2
2. On failure: abort and report error with stage `MANIFEST_RUNNING`
3. On success: extract output path (typically `ssot/tests/e2e/{proto_id}/e2e-coverage-manifest.yaml`)
4. Transition to `SPEC_RUNNING` state

### Phase 5: SPEC_RUNNING

1. Call Skill tool `/ll-qa-e2e-spec-gen` with:
   - `e2e_coverage_manifest_path`: path to the e2e-coverage-manifest.yaml from Phase 4
2. On spec-gen failure: log warning but preserve manifest (continue strategy per ADR-053 §2.3.2)
3. On success: verify at least one `JOURNEY-*.md` was generated
4. Transition to `COMPLETE` state

### Phase 6: COMPLETE

1. Compile execution summary with:
   - Paths to all generated artifacts
   - State machine final state
   - Any warnings (e.g., spec-gen continuation)
2. Return summary to user

## Error Handling

| Stage | Strategy | Action on Failure |
|-------|----------|-------------------|
| E2EPLAN_RUNNING | fail-fast | Abort, preserve no artifacts |
| TRACEABILITY_VALIDATING | fail-fast | Abort, preserve e2e-journey-plan |
| MANIFEST_RUNNING | fail-fast | Abort, preserve e2e-journey-plan |
| SPEC_RUNNING | continue | Log warning, preserve manifest |

## Constraints

- Do not directly generate artifacts — only call sub-skills via Skill tool
- Do not bypass acceptance traceability validation before manifest-init
- Do not modify artifacts generated by sub-skills
